# Step 1: 이름 짓기와 매직 넘버 (Naming & Magic Number)

이 코드는 **기능적으로 완벽하게 동작하며 모든 테스트를 통과**합니다.
하지만 코드를 처음 보는 동료 개발자(혹은 3개월 뒤의 나)는 이 코드가 도대체 무슨 일을 하는지 이해하기 매우 어렵습니다.

## 🎯 학습 목표
- 변수명과 메서드명에 의도를 담는 연습을 합니다.
- 코드 중간에 흩뿌려진 '의미를 알 수 없는 숫자(매직 넘버)'를 상수로 추출합니다.

## 📝 요구 사항
1. `BadUmpire.java` 코드를 열고, 메서드명(`doS`, `doB`)과 변수명(`a`, `b`, `r` 등)을 의미 있는 이름으로 변경하세요.
2. 코드 안에 하드코딩된 숫자들(예: `3`)의 의미를 파악하고, `private static final int`를 이용해 가독성 좋은 상수로 추출하세요.
3. **테스트 코드(`BadUmpireTest.java`)도 개선하세요.** "테스트는 문서다"라는 관점에서, 메서드 이름과 `@DisplayName` 모두 무엇을 검증하는지 알 수 있어야 합니다.
   - 메서드명: `t1` → `countStrike` (영어)
   - `@DisplayName`: `"테스트1"` → `"같은 자리에 같은 수가 있으면 스트라이크이다"` (한글 설명)
4. 리팩토링 중간중간 테스트를 실행(`gradle test`)하여, **기존의 동작이 깨지지 않았는지(Green)** 계속 확인하세요!

## ⌨️ IntelliJ 리팩토링 단축키

| 기능 | Windows/Linux | macOS |
|------|--------------|-------|
| **이름 변경** (Rename) | `Shift + F6` | `Shift + F6` |
| **상수 추출** (Extract Constant) | `Ctrl + Alt + C` | `⌘ + ⌥ + C` |
| **변수 추출** (Extract Variable) | `Ctrl + Alt + V` | `⌘ + ⌥ + V` |

> 💡 변수/메서드 위에 커서를 놓고 `Shift + F6`을 누르면 사용처까지 한번에 바뀝니다!
