코드가 스텝마다 계속 누적되는 하나의 프로젝트입니다.

> - **BE (Java):** IntelliJ에서 `part1-tdd-practice/java/` 폴더를 열어주세요.
> - **FE (TypeScript):** VSCode에서 `part1-tdd-practice/ts/` 폴더를 열고, `npm install`을 실행하세요.

TDD가 처음이라면 아래 문서를 먼저 읽어보세요. 숫자 야구를 예시로 TDD 흐름을 보여줍니다.
- 📌 [`TDD 예시`](https://www.notion.so/TDD-30fdcbbf080a80b1beb3ce850400a1cb?source=copy_link)

## 📌 목표
- Red-Green-Refactor 사이클을 몸으로 익힌다.
- 테스트하기 쉬운 도메인 객체를 설계한다.

## 🚫 규칙
1. **테스트 코드를 먼저 작성**(Red)할 것. 기능부터 짜지 마세요.
2. **Step별로 커밋**할 것. 커밋 메시지가 곧 당신의 사고 과정입니다.
3. `java/`와 `ts/` 디렉토리 각각에서 프로젝트를 하나씩 완성합니다.

---

## 🚀 진행 순서

### Step 1: 스트라이크 판정
- [ ] "같은 자리에 같은 수가 있으면 스트라이크" 로직만 TDD로 구현합니다.

### Step 2: 볼 판정 + 경계값
- [ ] 볼 판정 로직을 TDD로 추가합니다.
- [ ] **"3스트라이크인데 0볼"** 같은 엣지 케이스를 테스트로 검증하세요.

### Step 3: 숫자 생성 (랜덤의 격리)
- [ ] 1~9 사이 서로 다른 3자리 숫자를 생성하는 로직을 TDD로 구현합니다.
- [ ] 💡 랜덤은 테스트하기 어렵습니다. 이 "제어 불가능한 영역"을 인터페이스로 어떻게 분리할지 고민해보세요.

### Step 4: 게임 조립
- [ ] 판정, 숫자 생성, 콘솔 입출력을 조립하여 게임 루프를 완성합니다.
- [ ] 잘못된 입력(문자, 중복, 길이 초과)에 대한 예외 처리를 추가하세요.
- [ ] 게임 종료 후 재시작/완전종료 선택 기능을 구현합니다.
